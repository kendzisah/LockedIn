### `user_guild_ids()` helper (key for migration target)
Defined at `supabase/migrations/20260428013000_fix_guild_members_policy_recursion.sql:15`:
```sql
CREATE OR REPLACE FUNCTION public.user_guild_ids()
RETURNS SETOF uuid
LANGUAGE sql
SECURITY DEFINER
SET search_path = public
STABLE
AS $$
  SELECT guild_id
  FROM public.guild_members
  WHERE user_id = auth.uid();
$$;
REVOKE ALL ON FUNCTION public.user_guild_ids() FROM public;
GRANT EXECUTE ON FUNCTION public.user_guild_ids() TO authenticated;
```

## 4. Edge functions

### `complete-mission`
- **File**: `supabase/functions/complete-mission/index.ts`
- **Method/path**: POST `/functions/v1/complete-mission` (Supabase edge runtime). Handles `OPTIONS` preflight via shared CORS headers.
- **Request body (line 47–51)**:
  ```ts
  { focusMinutes: number, missionsDone: number, streakDays: number }
  ```
  Bounds (line 66–69): `0 ≤ focusMinutes ≤ 1440`, `0 ≤ missionsDone ≤ 50`, `0 ≤ streakDays ≤ 3650`. All must be finite numbers.
- **Response (line 125–127)**:
  ```ts
  { success: true, weekKey: string, totalScore: number,
    guilds: [{ guild_id: string, ok: boolean }] }
  ```
  Error shapes: 401 `{ error: 'Missing Authorization header' | 'Unauthorized' }`, 400 `{ error: 'Missing required fields ...' | 'Score values out of allowed range' }`, 500 `{ error: <message> }`.
- **Auth**: requires `Authorization: Bearer <jwt>` header from the mobile client; passes through to a Supabase client constructed with the anon key + caller JWT (line 78–85). Calls `auth.getUser()` to resolve `userId`.
- **Behavior**: computes `totalScore = focusMinutes*2 + missionsDone*15 + streakDays*10` server-side (line 17–19, line 98). Computes ISO week key from server UTC clock (line 22–30). For every row in `guild_members` for the user (line 101–104), calls RPC `upsert_guild_score` with caller's JWT (line 113–121).
- **Secrets read via `Deno.env.get(...)`**: `SUPABASE_URL`, `SUPABASE_ANON_KEY` (line 79–80). No service role key.
- **CORS** (`supabase/functions/_shared/cors.ts:1`):
  ```ts
  Access-Control-Allow-Origin: *
  Access-Control-Allow-Headers: authorization, x-client-info, apikey, content-type
  ```
- **Caller (mobile)**: `apps/mobile/src/features/leaderboard/GuildService.ts:530` (`GuildService.completeMissionServerSide`). Note the client sends an extra `timeGate` field in the body (line 531), but the server currently ignores it (the comment in the source labels it a stale client-only UX guard, not a security boundary).

There are no other edge functions in `supabase/functions/`.

  - `client.auth.getSession()` → if a session already exists, return its user ID.

- Fresh-install heuristic (`SupabaseService.ts:56-64`): if the AsyncStorage key `@lockedin/has_launched` is missing but a Keychain (SecureStore) session exists from a previous install, `client.auth.signOut()` is called first to clear it before the anonymous sign-in.
- `client.auth.onAuthStateChange` (line 69) updates the cached user ID on every auth event.

State produced: a `User` row in `auth.users` with `is_anonymous = true`; the `handle_new_user` trigger creates the corresponding `profiles` and (best-effort) `user_stats` rows.

### Email signup / signin / password reset
- **Signup** (`AuthService.ts:158-236`): if the current session is anonymous, signup falls through to `linkEmailPassword` (line 175–177). Otherwise calls `client.auth.signUp({ email, password })`. Detects duplicate-email-signup (line 199–207) where Supabase returns `200 + user with no identities`. When email confirmation is enabled, `session` is null and the user must confirm before sign-in.
- **Signin** (`AuthService.ts:242-282`): `client.auth.signInWithPassword({ email, password })`.
- **Reset password** (`AuthService.ts:448-472`): `client.auth.resetPasswordForEmail(email, { redirectTo: ENV.SUPABASE_PASSWORD_RESET_REDIRECT })`. Default redirect (no env override): `https://locked-in.co/auth/callback?next=/auth/reset-password` (`apps/mobile/src/config/env.ts:37`).
- **Change password** (`apps/mobile/src/features/settings/sheets/ChangePasswordSheet.tsx:48-53`): first re-verifies current password via `signInWithPassword`, then calls `client.auth.updateUser({ password: pw1 })`.

### Apple Sign-In
- Native flow via `expo-apple-authentication` (`AuthService.ts:288-362`):
  1. `AppleAuthentication.isAvailableAsync()`.
  2. `AppleAuthentication.signInAsync({ requestedScopes: [FULL_NAME, EMAIL] })` returns `credential.identityToken`.
  3. `client.auth.signInWithIdToken({ provider: 'apple', token: identityToken })`. Important comment at line 324–327: when an anonymous session is active, this same call **links** the Apple identity to the existing user (preserves UUID). The code does **not** use `linkIdentity` (that's for OAuth redirect, web).
- User-facing copy for cancellation: `MSG_APPLE_AUTH_FAILED = "There was a problem signing you in. Try again"` (line 27).

### Account linking (anonymous → email or Apple)
- **Email**: `AuthService.linkEmailPassword` (line 375-438). Calls `client.auth.updateUser({ email, password })`, then `auth.refreshSession()` so the `is_anonymous` claim flips. If `is_anonymous` is still true after refresh, the migration is treated as pending email confirmation and returns `MSG_CONFIRM_EMAIL_TO_FINISH`.
- **Apple**: `AuthService.linkAppleAccount` (line 444) delegates to `signInWithApple` — relying on Supabase's native ID-token flow to link rather than create.

### Sign-out
- `AuthService.signOut` (line 477-501): `client.auth.signOut()`, then `ensureAnonymousSession(client)` to immediately restore an anonymous session so the user stays in the app.

### Provider state
- `apps/mobile/src/features/auth/AuthProvider.tsx:53-96`. Subscribes via `AuthService.onAuthStateChange`. Exposes `{ user, isAuthenticated, isAnonymous, signUp, signIn, signInWithApple, linkAccount, linkAppleAccount, signOut, resetPasswordForEmail }`. Reactive updates to `Analytics.setIsAnonymous` and `Analytics.registerSuperProperties({ is_anonymous })` (line 63–66).

OAuth provider config (Apple) is set in Supabase dashboard outside the codebase — only the client-side token submission appears here.
